# OctoAcme Project Management Docs

This folder contains the process documentation for OctoAcme project management. Use it as the starting point for understanding how projects are run, who is responsible for what, and which artifacts to produce at each stage.

## Overview

OctoAcme follows a structured, iterative project management approach built on five core principles:

- **Customer-first** – prioritize customer value and usability in every decision.
- **Iterative delivery** – ship small, testable increments and learn continuously.
- **Clear ownership** – every project has a named Project Manager (PM) and Product Lead (PdM).
- **Data-informed decisions** – measure impact and iterate based on evidence.
- **Psychological safety** – encourage open feedback and a culture of learning.

Projects move through five lifecycle phases: **Initiation → Planning → Execution → Release → Retrospective**. Each phase has defined deliverables, decision gates, and communication expectations described in the documents below.
Welcome to the OctoAcme Project Management knowledge base. This repository documents how OctoAcme teams plan, execute, and continuously improve projects — aligned to customer-first and iterative delivery principles.

## Summary of OctoAcme Project Management Processes

OctoAcme follows a structured yet flexible approach to project management built around five core principles:

- **Customer-first**: Prioritize user value, outcomes, and usability at every stage of delivery.
- **Iterative delivery**: Break work into shippable increments, validate early, and adapt based on feedback.
- **Clear roles**: Project managers, product leads, developers, QA, and stakeholders each have defined responsibilities to reduce confusion and increase accountability.
- **Evidence-based decisions**: Use data, metrics, and retrospective learnings to guide planning and prioritization.
- **Open communication**: Regular syncs, project boards, and clear escalation paths keep all stakeholders aligned.

Projects move through a standard lifecycle: **Initiation → Planning → Execution → Release → Retrospective**, with risk management and clear role definitions supporting every phase.

## Getting Started

- **Approaching a release**: see [Release and Deployment](octoacme-release-and-deployment.md).
